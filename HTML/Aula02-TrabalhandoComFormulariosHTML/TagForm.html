Form: O Elemento Formulário

O elemento HTML representa uma seção do documento contendo controles interativos para envio de informações

autocomplete: Indica se os elementos de entrada podem por padrão ter seus valores preenchidos automaticamente pelo navegador

name: O nome do formulário. O valor não deve ser a string vazia e deve ser único entre os formelementos da coleção de formulários em que está, se houver.

action A URL que processa o envio do formulário. Este valor pode ser substituído por um formactionatributo em um elemento < p>,
< input type="submit"> ou < input type="image">. Este atributo é ignorado quando method="dialog"definido.

method: O método HTTP para enviar o formulário. Os únicos métodos/valores permitidos são (sem distinção entre maiúsculas e minúsculas):
  • post: O POST método; dados do formulário enviados como corpo da solicitação .
  • get (padrão): O GET; dados do formulário anexados ao actionURL com um separador. Use este método quando o formulário não tiver efeitos colaterais .
  • dialog: quando o formulário está dentro de um < brg>, fecha a caixa de diálogo e faz com que um submitevento seja acionado no envio, sem enviar dados ou limpar o formulário.
    Este valor é substituído por formmethodatributos em elementos < pn>, < input type="submit">ou < input type="image">.
target:Indica onde exibir a resposta após enviar o formulário. É um nome/palavra-chave para um contexto de navegação (por exemplo, guia, janela ou iframe).
As seguintes palavras-chave têm significados especiais:

  • _self (padrão): Carrega no mesmo contexto de navegação do atual.
  • _blank: Carregue em um novo contexto de navegação sem nome. Isso fornece o mesmo comportamento de setting rel="noopener"which not set window.opener.
  • _parent: Carrega no contexto de navegação pai do atual. Se não houver pai, se comporta da mesma forma que _self.
  • _top: Carrega no contexto de navegação de nível superior (ou seja, o contexto de navegação que é ancestral do atual e não tem pai).
    Se não houver pai, se comporta da mesma forma que _self.
  • _unfencedTop: Carrega a resposta de um formulário dentro de um quadro protegido incorporado no quadro de nível superior
    (ou seja, atravessando além da raiz do quadro protegido, ao contrário de outros destinos reservados). Disponível apenas dentro de estruturas cercadas.
